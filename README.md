**Проектный опыт**

**1. Хакатон Сеченовского Университета: Веб-платформа для прогнозирования характеристик опухоли молочной железы**
*Роль: Тимлид, full-stack разработчик*
*   Разработала архитектуру и прототип веб-приложения, интегрирующего модель машинного обучения с пользовательским интерфейсом.
*   Спроектировала и реализовала реляционную базу данных для хранения анонимизированных клинических данных пациентов.
*   Организовала работу команды: распределение задач, контроль сроков, интеграция компонентов (фронтенд, бэкенд, модель).
*   *Вызов и результат:* Столкнулись со сложностями интеграции модели, БД и веб-интерфейса в единый рабочий продукт, что потребовало корректировки плана и научило более точно оценивать временные затраты на интеграционные задачи.

**2. Школа 21: Медицинский портал (Full-stack проект)**
*Роль: Тимлид, основной разработчик*
*   Реализовала с нуля полнофункциональное веб-приложение для клиники.
*   **Backend (Django):** Спроектировала БД (3НФ), реализовала CRUD, REST API, JWT-аутентификацию с ролями, пагинацию, фильтрацию, экспорт/импорт данных.
*   **Frontend (React):** Создала административный интерфейс с динамическими таблицами, формами и графиками (Chart.js).
*   Организовала процесс разработки в команде из 3 человек, проводила код-ревью.

**3. Школа 21: Система мониторинга пациентов реанимации (Микросервисы)**
*Роль: Backend-разработчик в команде*
*   Участвовала в создании распределенной системы из 3 микросервисов (FastAPI) для эмуляции данных с медицинских датчиков.
*   Разрабатывала сервис-эмулятор пациента, генерирующий метрики (пульс, давление) и отправляющий их в Apache Kafka.
*   Работала над интерфейсом мониторинга (React + TypeScript) с графиками реального времени.

**4. Школа 21: Учебные проекты по базам данных и API**
*Роль: Индивидуальный разработчик*
*   Полный цикл: проектирование ER-диаграмм, создание и нормализация БД в Django, реализация REST API (DRF), настройка кеширования, логирования и написание юнит-тестов (Pytest).
